## Design Decisions

### Why replace /feature-dev instead of complementing it
The generic `/feature-dev` plugin (at `~/.claude/plugins/marketplaces/claude-plugins-official/plugins/feature-dev/`) has a 7-phase workflow with code-explorer, code-architect, and code-reviewer subagents. It doesn't know about Reelly's architecture rules, old-code policy, path aliases, or available MCPs. A project-tailored skill is more token-efficient and produces better results.

### Progressive disclosure (3 reference files, loaded on-demand)
- SKILL.md body loads when the skill triggers (~224 lines)
- References load only when their phase needs them
- This avoids dumping 600+ lines into context upfront

### Condensed references vs pointing to /docs/
The `/docs/` folder has ~3000 lines across 7+ files. We condensed the critical rules into ~160 lines in architecture-rules.md. This saves tokens per invocation while being self-contained. If docs change significantly, architecture-rules.md needs updating.

### Old code policy
Prominently placed in Phase 2 of SKILL.md as a boxed callout. Also enforced in the Phase 4 self-review checklist. The project has a flat legacy structure (`libs/hooks`, `libs/components`) while docs describe an aspirational 3D structure (`libs/listing-app/domain/`). Old code should not be refactored to match docs.

### Zustand over React Context
Added as a rule: Context re-renders all consumers on any change. Only acceptable for small isolated subtrees (2-3 direct children). Otherwise use Zustand.

### Size classification (S/M/L)
S tasks skip to Phase 3 (saves ~60% of tokens). M tasks do light discovery. L tasks run all phases. This prevents over-processing simple bug fixes.

## How to Update

### When /docs/ architecture rules change
Update `references/architecture-rules.md`. Re-read the changed docs and condense.

### When new patterns emerge in codebase
Update `references/patterns.md` with new file paths and code examples.

### When new MCPs or skills are installed
Update `references/tooling-guide.md` — add to the appropriate MCP or skill delegation table.

### When workflow needs adjustment
Edit SKILL.md directly. Keep under 400 lines. If approaching limit, move details to a reference file.
